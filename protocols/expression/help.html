• Use $X to get value of varbind with name X, e.g. $CPU.
    If name contain any special chars e.g.   or - then use square brackets $[X].
    To get value of non-current device use $[A => B], e.g. $[My switch #1 => CPU].
    Also you can use next phase
    • $X.value or $[X].value
    • $X.prev_value, $X.status, $X.value_type
    • $X.avg(10), $X.sum(10), $X.min(10) and $X.max(10) to get average, summa, minimun and maximum of last 10 (any) values
    • $X.forecast(10, 3) to get predicted value by linear regression based on prev 10 values for third future measurement.
    • $X.speed to get speed (per seconds) of value changing between measurements.
  • Use $[@PROP] to get device property, e.g. $[@status] or $[@latency].
  • Use $[A => @B] to get non-current device property, e.g. $[My switch #1 => @latency].
  • Values of these varbinds updated after each device poll.
  • You can use any valid js-expression, e.g:
    • ($CPU + $GPU)/2
    • Math.max($CPU, $GPU)
    • $TEXT.substr(1, 10)
    • $[Some varbind name].split(';')[0]
  • Save device and non-expression varbinds before use expressions.
